## 🎯 ПОЛНАЯ ИНСТРУКЦИЯ (3 терминала)

### ТЕРМИНАЛ 1: Flask сервер
```bash
python src/admin/app.py
```
Будет запущен на HTTPS: https://localhost:5000

### ТЕРМИНАЛ 2: Получить публичный URL
```bash
ssh -R 80:localhost:5000 serveo.net
```
Вы получите:
```
https://xxxxxxxx-xxx-xxx-xxx-xxxxxx.serveousercontent.com
```

**Скопируйте этот URL!** (он нужен для BotFather)

### ТЕРМИНАЛ 3: Запустить бота
```bash
# Сначала установите токен:
export TELEGRAM_BOT_TOKEN='YOUR_BOT_TOKEN_HERE'

# Затем запустите бота:
python src/bot/telegram_bot.py
```

## ⚙️ КОНФИГУРАЦИЯ BotFather

1. Откройте Telegram
2. Найти: @BotFather
3. Отправьте: `/setmenubutton`
4. Выберите вашего бота
5. Выберите "Web App"
6. **Введите URL:**
   ```
   https://xxxxxxxx-xxx-xxx-xxx-xxxxxx.serveousercontent.com/miniapp
   ```
   (Замените "xxxxx" на ваш URL из Терминала 2)

БОМ! Готово! 🎉

## 🐛 ЕСЛИ ЧТО-ТО НЕ РАБОТАЕТ

**Проблема:** "URL is invalid (https is required)"
→ Используйте публичный URL из serveo, не localhost

**Проблема:** Миниапп не загружается
→ Проверьте: `python test_flask.py`
→ Должны быть ✓ по всем пунктам

**Проблема:** Бот не запускается
→ Проверьте: `export TELEGRAM_BOT_TOKEN=your_token`
→ Проверьте логи: `logs/bot.log`

**Проблема:** Serveo URL не работает после перезагрузки
→ Это нормально, serveo дает новый URL каждый раз
→ Просто обновите в BotFather новый URL

---

## 💡 СОВЕТЫ

- Flask URL должна быть HTTPS (не HTTP!)
- Serveo дает временный URL (меняется при переподключении)
- Для постоянного URL используйте ngrok с токеном
- Самоподписанный сертификат OK для разработки
- Миниапп синхронизация происходит автоматически через Telegram API
