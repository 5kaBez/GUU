# 📚 Schedule Admin Panel

Панель администратора для управления расписанием университета.

## ✨ Что включено

- **Загрузка расписания** - импорт расписания из Excel/CSV файлов
- **Просмотр расписания** - таблица с возможностью фильтрации:
  - По программе (ищет в названии)
  - По институту
  - По направлению подготовки
  - По дню недели
  - По курсу
  - Сортировка по программе или времени
- **Резервные копии** - создание и восстановление backups БД
- **Просмотр пользователей** - статистика использования
- **Логирование** - все операции логируются

## 📋 Требования к Excel файлу

Необходимые колонки в Excel:

1. **Форма обучения** - Очная/Заочная
2. **Уровень образования** - Бакалавриат/Магистратура
3. **Курс** - 1, 2, 3, 4 (цифра)
4. **Институт** - название института
5. **Направление** - направление подготовки
6. **Программа** (или **Группа**) - программа обучения (парсер автоматически переименует "Группа" в "Программа")
7. **Номер группы** - номер группы студентов (1, 2, 3...)
8. **День недели** - Понедельник, Вторник, ... Воскресенье
9. **Номер пары** - 1, 2, 3, 4... (цифра)
10. **Время пары** - время начала-конца (например: 09.00-10.30)
11. **Чётность** - 0 (нечётная) или 1 (четная неделя)
12. **Предмет** - название предмета **[ОБЯЗАТЕЛЬНО]**
13. **Вид пары** - Лекция/Практика/Семинар
14. **Преподаватель** - ФИО преподавателя
15. **Номер аудитории** - номер аудитории
16. **Недели** - номера недель (например: 1-4, 5-16)

> ⚠️ **Важно**: Колонна "Предмет" обязательна! Строки без предмета будут пропущены.

## 🚀 Запуск


## 📊 Функции

### 1. Загрузка расписания
- Поддерживаемые форматы: .xlsx, .xls, .csv
- Два режима загрузки:
  - **Replace**: полностью переписать расписание
  - **Append**: добавить к существующему расписанию
- Автоматическое логирование при ошибках
- Если колонна "Программа" отсутствует или пуста, создается/заполняется значениями

### 2. Просмотр расписания
- Таблица со всеми парами
- Фильтры:
  - **Поиск программы**: поиск по названию программы
  - **Институт**: фильтр по названию института
  - **Направление**: фильтр по направлению (зависит от выбранного института!)
  - **День недели**: только выбранный день
  - **Курс**: только выбранный курс
- Сортировка:
  - По программе (А-Я или Я-А)
  - По времени пары (рано-поздно или поздно-рано)
- Паджинация: по 50 записей на странице

### 3. Резервные копии
- Кнопка "Создать резервную копию" - сохраняет snapshot БД
- Кнопка "Очистить данные" - удаляет все расписание
- Все backups в папке `data/backups/`

### 4. Пользователи
- Просмотр всех зарегистрированных пользователей
- DAU (Daily Active Users) - активные сегодня
- MAU (Monthly Active Users) - активные за 30 дней
- История для каждого пользователя

## 📝 Логирование

Логи сохраняются в папке `logs/`:

- **admin_app.log** - основные логи админ панели
- **parser.log** - логи импорта расписания

Используйте логи для отладки проблем при загрузке файлов.

## 🐛 Решение проблем

**Проблема**: Колонна "Программа" показывает прочерки при просмотре

**Решение**: Парсер автоматически:
1. Переименует "Группа" → "Программа" если "Программа" не существует
2. Заполнит пустые значения "Программа" значениями из колонны "Направление"
3. Если и "Направление" пустое, заполнит значением "-"

**Проблема**: Направления не фильтруются по институту

**Решение**: В селекторе "Направление" автоматически показываются только направления, которые есть в выбранном институте

**Проблема**: Импорт возвращает ошибку

**Решение**: Проверьте файл `logs/parser.log` для деталей ошибки

## 📞 Поддержка

Вся логика фильтрации и поиска находится в:
- Backend: `src/admin/app.py` (routes с `/api/`)
- Frontend: `src/admin/static/dashboard.js` (функции loadSchedule, updateDirectionFilter)
- БД: `data/schedule.db` (таблица schedule)

Для примера SQL запросов смотрите `src/admin/app.py`.
